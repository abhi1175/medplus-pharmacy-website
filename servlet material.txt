A Java Servlet is a server-side technology used to create dynamic web applications.
It runs inside a web container (like Apache Tomcat) and handles requests from clients (usually web
browsers),
processes them, and generates responses, typically in HTML.

Servlets are an integral part of Java EE (Jakarta EE) and are widely used to build scalable web
applications.

Servlet Lifecycle
The lifecycle of a servlet is managed by the servlet container and includes the following stages:
1. Loading and Instantiation - The container loads the servlet class and creates an instance.
2. Initialization (init) - The servlet calls the init() method once during initialization.
3. Request Handling (service) - The service() method is called for each client request.
4. Destruction (destroy) - The destroy() method is called before the servlet is removed from memory.

Example: Basic Java Servlet
import java.io.*;
import javax.servlet.*;
import javax.servlet.http.*;

public class HelloServlet extends HttpServlet {
public void doGet(HttpServletRequest request, HttpServletResponse response)
throws ServletException, IOException {
response.setContentType("text/html");
PrintWriter out = response.getWriter();
out.println("<h1>Hello, Welcome to Java Servlets!</h1>");
}
}

Deploying the Servlet
1. Compile the Java file and place the class file in WEB-INF/classes directory.
2. Define servlet in web.xml:

<servlet>
<servlet-name>HelloServlet</servlet-name>
<servlet-class>HelloServlet</servlet-class>
</servlet>

<servlet-mapping>
<servlet-name>HelloServlet</servlet-name>
<url-pattern>/hello</url-pattern>
</servlet-mapping>

3. Deploy the web application on Tomcat and visit http://localhost:8080/YourApp/hello
